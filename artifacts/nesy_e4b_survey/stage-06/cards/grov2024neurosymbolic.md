# On the Use of Neurosymbolic AI for Defending Against Cyber Attacks

*source: arxiv_html | tokens: 13071*

This paper, "On the Use of Neurosymbolic AI for Defending Against Cyber Attacks," functions as a methodological proposal, arguing for the necessity of combining connectionist (ML) and symbolic (reasoning) AI—Neurosymbolic AI (NeSy)—to enhance Security Operations Center (SOC) capabilities. The core problem addressed is the inherent insufficiency of purely preventive or purely data-driven detection methods against modern, evolving cyber threats. The authors structure their argument around the MAPE-K incident management framework, identifying nine critical challenges across the Monitor, Analyse, Plan, and Execute phases.

The technical approach is highly systematic: first, they categorize existing AI challenges (e.g., Challenge 1: achieving optimal ML accuracy under real-world concept drift; Challenge 3: extracting and integrating Cyber Threat Intelligence (CTI) into ML models). They then propose ten specific NeSy use cases, ranging from using **Logic Tensor Networks (LTN)** to constrain ML detectors (Use Case 1) to employing **LLMs** for hypothesis generation in threat hunting (Use Case 3).

The feasibility of this approach is demonstrated through two proof-of-concept experiments. The first uses **LTN** to improve intrusion detection. By encoding domain knowledge (e.g., axioms defining non-web traffic) into the model, the authors report a near doubling of precision compared to a baseline fully connected neural network when classifying traffic from a subset of the **CICIDS2017 dataset** using **NetFlow entries**. The second experiment tackles situational awareness (Use Case 8) by combining LLMs and symbolic reasoning. Here, **GPT-4** is used to translate natural language CTI reports into **Linear Temporal Logic ($\text{LTL}_f$)** formulas, which are then processed by an **ASP solver (telingo)** to postdict if observed alerts (simulated from a SIEM system) align with known adversary attack patterns derived from the **MITRE ATT&CK** framework.

The paper's primary contribution is defining a structured set of NeSy use cases for the SOC field, suggesting that NeSy is ideal because it allows systems to be human-auditable, learn from limited incident data, and handle concept drift. Limitations highlighted include the scarcity of realistic, labeled datasets that contain both raw events and necessary symbolic knowledge, suggesting future work must focus on developing "NeSy datasets" to move beyond simplified PoCs.
