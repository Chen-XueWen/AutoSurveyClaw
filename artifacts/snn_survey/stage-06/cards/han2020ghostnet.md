# GhostNet: More Features From Cheap Operations

*source: s2_fulltext | tokens: 9748*

Han et al. (2020) from Huawei Noah’s Ark Lab propose GhostNet to address the computational inefficiency of deploying Convolutional Neural Networks (CNNs) on embedded devices. The authors identify a key characteristic of successful CNNs: significant redundancy in feature maps, where many output channels are linearly related to a smaller subset of intrinsic features. Unlike prior work that focuses on post-hoc compression (pruning, quantization) or architectural units like depthwise separable convolutions (MobileNet, ShuffleNet), GhostNet explicitly exploits this redundancy during architecture design to generate more features from cheap operations.

The core contribution is the Ghost Module, which decomposes a standard convolutional layer into two stages. First, a primary convolution generates a small number of intrinsic feature maps. Second, a series of cheap linear transformations, specifically depthwise convolutions with kernel sizes 3x3 or 5x5, are applied to these intrinsic maps to generate the remaining "ghost" feature maps. This design allows the network to produce the same number of output channels as a standard convolution but with significantly reduced parameters and FLOPs. The authors stack these modules into Ghost Bottlenecks, creating the GhostNet architecture, which mirrors MobileNetV3 but utilizes Ghost modules for feature extraction. The method is compatible with standard CNN backbones and does not require automatic search, making it a practical solution for mobile vision tasks.

Extensive experiments validate the approach across classification and detection tasks. On the ImageNet ILSVRC-2012 classification benchmark, GhostNet (1.3× width) achieves 75.7% Top-1 accuracy with 226M FLOPs, surpassing MobileNetV3 Large (75.2% accuracy, 219M FLOPs) and EfficientNet. On CIFAR-10, Ghost-VGG-16 (s=2) attains 93.7% accuracy with approximately half the FLOPs of the original VGG-16 (93.6% accuracy). For object detection on MS COCO, GhostNet backbones integrated into RetinaNet and Faster R-CNN frameworks achieve 26.6% mAP with 164M FLOPs, outperforming MobileNetV3 backbones which require 219M FLOPs for 26.4% mAP.

Limitations include the necessity of tuning hyperparameters s (expansion ratio) and d (linear kernel size) to balance accuracy and efficiency. While the linear operations are depthwise convolutions rather than truly parameter-free shifts, the method offers a distinct alternative to NAS-based efficient designs. While primarily a CNN efficiency study, the concept of generating feature sets from intrinsic subsets offers insights for architectures processing feature ensembles. Code is available at the authors' GitHub repository.
