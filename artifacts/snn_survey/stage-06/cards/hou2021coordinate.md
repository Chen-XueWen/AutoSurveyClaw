# Coordinate Attention for Efficient Mobile Network Design

*source: s2_fulltext | tokens: 9623*

Hou et al. introduce Coordinate Attention (CA), a lightweight attention mechanism designed to enhance the performance of mobile networks by integrating positional information into channel attention. The authors address a critical limitation in existing attention modules for resource-constrained vision tasks: standard Squeeze-and-Excitation (SE) blocks model inter-channel dependencies but discard spatial positional information via 2D global pooling, while spatial attention methods like CBAM rely on convolutions that capture only local relations. CA resolves this by factorizing channel attention into two parallel 1D feature encoding processes. Instead of a single global descriptor, the input feature map is pooled along the horizontal and vertical axes separately, generating direction-aware feature maps that embed spatial coordinates into the channel attention. These are then transformed into position-sensitive attention maps that preserve precise location data while capturing long-range dependencies along one spatial direction.

Extensive experiments validate CA’s efficacy across classification, detection, and segmentation. On ImageNet, integrating CA into MobileNetV2 yields a Top-1 accuracy of 74.3%, surpassing the baseline (72.3%) by 2.0% and outperforming SE (73.5%) and CBAM (73.6%) with nearly identical parameter counts (3.95M vs 3.89M). Performance gains are consistent across network scales; MobileNetV2-0.5 improves from 65.4% to 67.0%. When applied to EfficientNet-b0, CA achieves 76.9% Top-1 accuracy, exceeding the SE-based variant (76.3%).

The mechanism demonstrates superior transferability to downstream tasks requiring spatial precision. In object detection on the COCO validation set using SSDLite320, CA improves AP from 22.3 to 24.5, outperforming SE and CBAM variants. For semantic segmentation on Pascal VOC 2012, MobileNetV2 equipped with CA reaches 73.32% mIoU (stride 16), significantly higher than SE (71.69%) and CBAM (71.28%). Similar improvements are observed on Cityscapes (74.0% mIoU). While the method incurs a marginal latency increase (17-19ms vs 14-16ms on Google Pixel 4) and computational overhead (310M M-Adds vs 300M), the trade-off favors accuracy, particularly for dense prediction tasks where coordinate awareness is crucial. The authors note that CA is compatible with standard mobile blocks like inverted residuals and sandglass bottlenecks, offering a simple plug-in alternative to SE for enhancing spatial representation in resource-constrained environments. Unlike set-based architectures that process unordered data, CA explicitly leverages the grid structure of images, making it distinctively suited for convolutional backbones where spatial locality is inherent.
